## <a name="changes-coming-to-partner-center-apis"></a><a name="9"></a>Ändringar kommer till Partner Center-API:er

### <a name="categories"></a>Kategorier

- Datum: 2020-07-28
- Funktioner

### <a name="impacted-audience"></a>Påverkad målgrupp

Direktfaktureringspartner och indirekta leverantörer som deltar i CSP-programmet som använder Partner Center-API:erna. 

### <a name="summary"></a>Sammanfattning

Partnercenter-teamet har utvärderat de exponerade Partner Center-API:erna. Vi ser att vissa av dessa API:er är inaktuella och inte längre stöds. De dras tillbaka den 31 augusti 2020.

### <a name="details"></a>Information

CSP Partner Center-teamet har identifierat några API:er som vi planerar att dra tillbaka i slutet av 31 augusti 2020. Dessa API:er representerar gränssnitt som utformades för flera år sedan, och många av dem fungerar inte längre. Teamet håller på att ta bort referenser till dessa slutpunkter i nästa SDK-version under det sista kalenderkvartet 2020. Nedan visas de API:er som ska inaktiveras. Partner bör se till att de inte refererar till dessa URI:er i framtiden.

**Partnertjänstbegäranden** – Dessa slutpunkter gjorde det möjligt för en partner att hämta supportämnessträngar och sedan skicka partnertjänstbegäranden programmatiskt. Med tiden blev det här API:et instabilt och stöds inte. Partner dirigeras till partnercentrets användargränssnitt för att skapa partnersupportärenden. Användarupplevelsen ger partner ytterligare information när de skapar supportärenden, till exempel rekommenderade steg och dokument för det område som de har problem med. Partnercenter-teamet har också förbättrat skapandet av partnerbiljetten genom att anpassa formulären för tjänstbegäran för att samla in information som är specifik för problemområdet. Informationen som samlas in av onlineformulären hjälper supporttekniker att snabbt och korrekt lösa dina problem. Dessa URI:er ska inte refereras till eller användas och tas ur drift i slutet av augusti 2020. Om du inser att dessa ändringar  endast påverkar supportämnena och skapar partnertjänstbegärandemetoder bör alla andra supportrelaterade API:er fortsätta att fungera som förväntat.

- {base-URL}/v1/servicerequests/supporttopics
- {base-URL}/v1/servicerequests/{agent-locale}

**Office tjänstincidenter** – Dessa slutpunkter har utformats för att göra det möjligt för en partner att få en samlad vy över tjänstincidenter/avbrott i alla partnertjänster. De data som dessa slutpunkter returnerar har dock blivit inaktuella och inte användbara. Vi kommer att inaktivera dessa slutpunkter i slutet av juli. I framtiden kommer Partner Center-teamet att omvärdeera scenarierna för alla kunders incidenter och hälsotillstånd för tjänster och leta efter möjligheter att ge mervärde inom det här området. Referenser till dessa URI:er bör tas bort.

- {base-URL}/v1/serviceincidents/office/count
- {base-URL}/v1/serviceincidents/office

Partner kan använda [Office 365 Service Communications-API:et](/office/office-365-management-api/office-365-service-communications-api-reference) för att få uppdaterad status för kundtjänster.

**TopProduct** – den här slutpunkten skapades som ett konceptbevis som aldrig var avsett för allmän tillgänglighet. Den har utformats för att göra det möjligt för partner att hantera de främsta erbjudandena som skulle återges i Partner Center-katalogen. Det här API:et har dock aldrig utvecklats fullständigt och kommer att inaktiveras den 31 juli 2020. Partner bör ta bort alla referenser till den.

- {base-URL}/v1/preferences/catalog/products

### <a name="next-steps"></a>Nästa steg

- Partner bör se till att deras lösningar inte längre refererar till ovanstående API:er.
- Molnlösningsleverantör kan besöka communityn [för Partnercenter-SDK API-Yammer för](https://www.yammer.com/cloudpartnercommunity/#/Threads/show?threadId=786951767490560) att ställa frågor om ytterligare klargöranden behövs.

## <a name="sdk-release-on-net-standard-v1161-preview"></a><a name="8"></a>SDK-version på .NET Standard (v1.16.1-preview)

### <a name="categories"></a>Kategorier

- Datum: 2020-07-21
- Funktioner

### <a name="impacted-audience"></a>Påverkad målgrupp

Direktfaktureringspartner och indirekta leverantörer som deltar i CSP-programmet som använder Partner Center .NET SDK. 

### <a name="summary"></a>Sammanfattning

Från och med 21 juli 2020 kan partner börja ladda ned förhandsversionen av **.NET Standard SDK Microsoft Partner Center** .NET SDK v1.16.1-preview tillsammans med uppdaterade offentliga **Partnercenter-SDK GitHub-exempel.**

### <a name="details"></a>Information

Direktfaktureringspartner och indirekta leverantörer som deltar i CSP-programmet är berättigade att ladda ned den senaste förhandsversionen av .NET Standard SDK. Vi kommer att migrera Microsoft Partnercenter-SDK från .NET Framework till .NET Standard 2.0-plattformen. Dessa ändringar gör SDK kompatibel med det befintliga programmet med hjälp av .NET Framework 4.6.1 och högre. .NET Standard stöder .NET Core 2.0 och högre.

Det har funnits en enorm efterfrågan från partner som begär SDK i .NET Standard. Den aktuella versionen av SDK tvingar partner att använda program som skrivits i ASP.NET Framework, vilket är långsamt och dyrt jämfört med .NET Core-program. Migreringen till .NET-standarden kan därför hjälpa till att åtgärda problem med prestanda eller kompatibilitet med .NET Core.
 
**Höjdpunkter**
- Inga ytterligare versioner från Microsoft för .NET Framework 4.8 eftersom det är den sista i raden.
- Begränsar framtida utveckling eftersom de flesta bibliotek kommer att släppas för .NET Core- eller .NET-standarden, som kanske inte stöder den aktuella ramverksversionen.
- .NET Core är fördelaktigt att använda med containrar på grund av dess lätta natur.
- Högpresterande serverkörning för Windows Server och Linux gör .NET Core till ett webbramverk med bäst prestanda.

### <a name="next-steps"></a>Nästa steg

- Ladda ned den senaste versionen av [.NET SDK v1.16.1-preview](https://www.nuget.org/packages/Microsoft.Store.PartnerCenter/1.16.1-preview)
- Version: .NET SDK v1.16.1-preview 
- Exempelkod: [Exempelkod](https://github.com/microsoft/Partner-Center-DotNet-Samples)
- Förhandsversion tillgänglig den: 2020-07-21 
- GA: Tre veckor från förhandsversionen (för att ge tillräckligt med tid för partner att migrera) 
- Det finns inga nya API-ändringar som en del av den aktuella versionen

## <a name="perpetual-software-in-the-cloud-solution-provider-program"></a><a name="5"></a>Permanent programvara i Molnlösningsleverantör program 


### <a name="categories"></a>Kategorier

- Datum: 2020-07-15
- Erbjudanden/marknader
- Status: Tillgänglig för en uppsättning kvalificerade indirekta leverantörer och deras indirekta återförsäljare. Allmänt tillgänglig i januari 2021.
 
 
### <a name="impacted-audience"></a>Målgrupp som påverkas

Indirekta CSP-återförsäljare och kvalificerade CSP Indirect-leverantörer 

### <a name="details"></a>Information

På Microsoft arbetar vi för att stödja alla organisationer med deras resa mot digital omvandling. Vi gör detta genom att inte bara investera i teknik, utan även genom att undersöka kund- och partnerresan från att skaffa vår teknik till deras supportupplevelse. I det här arbetet har vi identifierat en möjlighet att omvandla oss själva – genom att investera i vår handelsplattform.  Vi förenklar också licensieringsprogram samtidigt som vi utvecklar den nya handelsupplevelsen så att vi kan förbättra hur vi ger våra partner fler möjligheter. 

I och med att vi förbättrar våra handelsfunktioner för att ge kunder och partner en bättre köpupplevelse utökar vi erbjudandeportföljen så att den omfattar tillgängligheten av permanenta programvarulicenser i Molnlösningsleverantör-programmet (CSP). Genom att lägga till programvarulicenser i CSP kan deltagande partner fokusera på att erbjuda olika lösningar och hanterade tjänster till sina kunder. Det ger också kundernas övergång till molnet och ger partner den flexibilitet som krävs för hybridmolnmiljöer.   

Från och med 1 juli ökar vi den här funktionen genom en uppsättning kvalificerade indirekta leverantörer och deras indirekta återförsäljare som kommer att kunna utföra licenser för lokal programvara med kunder i CSP-programmet. Kvalificerade partner hanteras av Microsoft och måste vara både aktiva indirekta CSP-leverantörer och Microsoft-auktoriserade distributörer för open license-programmet som körs i ett geografiskt område som för närvarande stöds för försäljning av programvara inom CSP. Från och med januari 2021 skalar Microsoft den här funktionen ytterligare och gör det möjligt för alla CSP-partner, indirekta leverantörer och indirekta återförsäljare att sälja programvarulicenser i CSP-programmet.
